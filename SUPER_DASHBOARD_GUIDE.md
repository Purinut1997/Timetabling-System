# Super Admin Dashboard - คู่มือการใช้งาน

## ภาพรวมระบบ

Super Admin Dashboard เป็นหน้าจอหลักสำหรับผู้ดูแลระบบระดับสูงสุด สามารถจัดการทุกอย่างในระบบ Timetabling System ได้

---

## 🏠 ภาพรวม (Overview)

### ฟังก์ชันการทำงาน:
- **แสดงสถิติสำคัญ:** จำนวนโรงเรียน, ผู้ใช้, การสอนแทน, บัญชีรอจัดการ
- **การกระทำเร็ว:** เพิ่มโรงเรียน, เพิ่มผู้ใช้, รีเฟรชข้อมูล, ส่งออกรายงาน
- **การแสดงผลแบบ Real-time:** ข้อมูลอัพเดทจากฐานข้อมูลจริง

### คุณสมบัติพิเศษ:
- **Glassmorphism Design:** การออกแบบแบบกระจกเรืองแสงทันสมัย
- **Gradient Effects:** การใช้สีไล่ระดับเพื่อความสวยงาม
- **Responsive Layout:** รองรับทุกขนาดหน้าจอ
- **Interactive Animations:** การเคลื่อนไหวที่สมูทและน่าดึงดูด

## 🏫 จัดการโรงเรียน (Schools Management)

### ฟังก์ชันการทำงาน:

#### 1. **ดูรายการโรงเรียนทั้งหมด**
- แสดงชื่อโรงเรียน, ที่อยู่, อีเมลติดต่อ
- แสดงวันที่สร้างและอัพเดทล่าสุด
- สามารถค้นหาโรงเรียนได้

#### 2. **เพิ่มโรงเรียนใหม่**
- กรอกข้อมูล: ชื่อโรงเรียน, ที่อยู่, อีเมลติดต่อ
- ระบบตรวจสอบความถูกต้องของข้อมูล
- บันทึกลงฐานข้อมูลทันที

#### 3. **แก้ไขข้อมูลโรงเรียน**
- คลิกปุ่มแก้ไขข้อมูลโรงเรียนที่ต้องการ
- แก้ไขข้อมูลได้ทุกฟิลด์
- บันทึกการเปลี่ยนแปลงทันที

#### 4. **ลบโรงเรียน**
- ตรวจสอบว่าโรงเรียนไม่มีผู้ใช้อยู่
- ยืนยันการลบก่อนดำเนินการ
- ลบข้อมูลถาวรจากระบบ

### API Endpoints:
- `GET /super-schools` - ดูโรงเรียนทั้งหมด
- `POST /super-schools` - เพิ่มโรงเรียนใหม่
- `PUT /super-schools` - อัพเดทข้อมูลโรงเรียน
- `DELETE /super-schools/{id}` - ลบโรงเรียน

## 👥 จัดการผู้ใช้ทั้งหมด (Users Management)

### ฟังก์ชันการทำงาน:

#### 1. **ดูรายการผู้ใช้**
- แสดงชื่อ, อีเมล, บทบาท, สถานะ, โรงเรียน
- กรองตามโรงเรียน, บทบาท, สถานะ
- ค้นหาผู้ใช้ตามชื่อหรืออีเมล

#### 2. **เพิ่มผู้ใช้ใหม่**
- เลือกบทบาท: Super Admin, Admin, Teacher
- กำหนดโรงเรียนที่สังกัด
- ตั้งรหัสผ่านเริ่มต้น
- สถานะเริ่มต้น: Active

#### 3. **แก้ไขข้อมูลผู้ใช้**
- เปลี่ยนชื่อ, อีเมล, บทบาท, สถานะ
- ย้ายโรงเรียนสังกัด
- รีเซ็ตรหัสผ่าน

#### 4. **จัดการสถานะ**
- Active: ใช้งานได้ปกติ
- Inactive: ระงับการใช้งานชั่วคราว
- Suspended: ระงับการใช้งานถาวร

#### 5. **รีเซ็ตรหัสผ่าน**
- สร้างรหัสผ่านใหม่
- ส่งอีเมลแจ้งผู้ใช้
- บังคับให้เปลี่ยนรหัสผ่านครั้งแรก

### ข้อจำกัด:
- ไม่สามารถลบ Super Admin ได้
- ไม่สามารถเปลี่ยนบทบาทของตนเองได้
- อีเมลต้องไม่ซ้ำกันในระบบ

### API Endpoints:
- `GET /super-users` - ดูผู้ใช้ทั้งหมด (พร้อมกรอง)
- `POST /super-users` - เพิ่มผู้ใช้ใหม่
- `PUT /super-users` - อัพเดทข้อมูลผู้ใช้
- `DELETE /super-users/{id}` - ลบผู้ใช้
